1. Цель

Пользователь входит в учётную запись qwertystock.com в браузере на материнском сайте. После входа браузер возвращается в локальный кабинет (http://127.0.0.1:7332 / http://localhost:7332), который получает токены и может показывать профиль и вызывать API.

Отдельного пароля в локальном приложении нет: OAuth 2.0 Authorization Code + PKCE, публичный клиент, без client_secret в бинарнике.

6. Поток (Authorization Code + PKCE)

6.1 Подготовка (локальный кабинет)

  1. Случайный code_verifier (43–128 символов: [A-Za-z0-9-._~]).
  2. code_challenge = BASE64URL(SHA256(code_verifier)) без padding =.
  3. Случайный state (≥128 бит энтропии), сохранить до callback.

6.2 Редирект на материнский сайт (GET)

https://www.qwertystock.com/oauth/authorize ?response_type=code &client_id=qwertystock-local-cabinet &redirect_uri=<URL-encoded> &scope=openid%20profile%20email &state=<state> &code_challenge=<code_challenge> &code_challenge_method=S256

redirect_uri — один из зарегистрированных. Если пользователь не залогинен — форма входа на домене www.qwertystock.com.

6.3 Callback

http://127.0.0.1:7332/auth/callback?code=<authorization_code>&state=<state>

Проверка state, отклонение при несовпадении или без code.

6.4 Обмен кода на токены

POST https://www.qwertystock.com/oauth/token, Content-Type: application/x-www-form-urlencoded

ПараметрЗначение
grant_typeauthorization_code
codeИз callback
redirect_uriТот же, что в authorize
client_idqwertystock-local-cabinet
code_verifierИз шага 6.1

Пример ответа (JSON):

{ "access_token": "...", "token_type": "Bearer", "expires_in": 3600, "refresh_token": "...", "id_token": "..." }

При scope openid — id_token обязателен (проверка JWT по JWKS, aud = client_id, iss = issuer). Ошибки — RFC 6749 (error, error_description).

6.5 UserInfo (OIDC)

GET .../oauth/userinfo с Authorization: Bearer <access_token>.

7. CORS

Редиректы authorize/callback CORS не требуют. POST /oauth/token с сервера локального кабинета — CORS не нужен. Если токен запрашивает только JS на localhost к API материнского сайта — понадобится явный CORS (отдельное решение).

8. Безопасность

  1. PKCE обязателен.
  2. state обязателен (CSRF).
  3. redirect_uri только whitelist.
  4. Не передавать client_secret в локальный клиент.
  5. HTTPS на материнском сайте для authorize/token.
  6. HTTP localhost для redirect допустим (loopback, RFC 8252).

9. Чеклист для разработчика материнского сайта

  • Клиент qwertystock-local-cabinet (public, PKCE S256).
  • Whitelist redirect URI (оба localhost).
  • GET /oauth/authorize, POST /oauth/token.
  • grant_type=authorization_code + code_verifier.
  • Желательно OIDC: id_token, userinfo, discovery, JWKS.
  • Документация ошибок и TTL токенов.

10. Версионирование

manifestVersion в JSON — версия контракта. Смена эндпоинтов или client_id — новая версия манифеста и согласование с командой кабинета.

11. English summary

QwertyStock Local Cabinet (FastAPI on http://127.0.0.1:7332) uses OAuth 2.0 Authorization Code with PKCE against www.qwertystock.com. Register public client qwertystock-local-cabinet, whitelist the two localhost redirect URIs, implement authorize and token endpoints; optionally full OIDC (userinfo, id_token, discovery, JWKS). No client secret. Canonical URLs are in the JSON manifest above.
